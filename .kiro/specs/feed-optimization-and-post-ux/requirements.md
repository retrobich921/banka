# Requirements Document

## Introduction

Данный документ описывает требования к улучшению функционала постов в приложении Banka. Фокус на двух ключевых областях: оптимизация главной страницы (ленты постов) для отображения всех постов из групп пользователя с эффективной загрузкой, и улучшение пользовательского опыта создания поста с автоматическим открытием камеры, кропом 1:1 и запоминанием последней выбранной группы.

## Glossary

- **Feed_System**: Система отображения ленты постов на главной странице приложения
- **Post_Creation_System**: Система создания нового поста-банки
- **Camera_Module**: Модуль работы с камерой устройства для захвата фото
- **Group_Memory_Service**: Сервис для сохранения и восстановления последней выбранной группы пользователя
- **User**: Авторизованный пользователь приложения
- **Group**: Группа/коллекция, к которой принадлежит пользователь
- **Post**: Пост-банка с фотографиями и метаданными
- **Pagination**: Механизм постраничной загрузки данных
- **Lazy_Loading**: Механизм отложенной загрузки данных по мере необходимости

## Requirements

### Requirement 1: Unified Feed Display

**User Story:** Как пользователь, я хочу видеть все посты из всех моих групп на главной странице, чтобы не пропускать новый контент от сообществ, в которых я состою.

#### Acceptance Criteria

1. WHEN THE User открывает главную страницу, THE Feed_System SHALL загрузить посты из всех групп, в которых состоит User
2. THE Feed_System SHALL объединить посты из всех групп User в единую ленту
3. THE Feed_System SHALL отсортировать объединённые посты по дате создания в порядке убывания
4. WHEN THE Feed_System загружает посты из нескольких групп, THE Feed_System SHALL выполнять запросы параллельно
5. THE Feed_System SHALL отображать посты в хронологическом порядке независимо от группы происхождения

### Requirement 2: Feed Pagination

**User Story:** Как пользователь, я хочу, чтобы лента загружалась быстро и не тормозила приложение, чтобы комфортно просматривать контент даже при большом количестве постов.

#### Acceptance Criteria

1. THE Feed_System SHALL загружать посты порциями по 20 элементов
2. WHEN THE User прокручивает ленту до конца текущей порции, THE Feed_System SHALL автоматически загрузить следующую порцию постов
3. THE Feed_System SHALL отображать индикатор загрузки при получении следующей порции данных
4. THE Feed_System SHALL кэшировать загруженные посты для предотвращения повторных запросов
5. WHEN THE Feed_System загружает следующую порцию, THE Feed_System SHALL использовать курсор последнего загруженного поста
6. IF загрузка следующей порции завершается ошибкой, THEN THE Feed_System SHALL отобразить сообщение об ошибке и предложить повторить попытку

### Requirement 3: Camera-First Photo Capture

**User Story:** Как пользователь, я хочу, чтобы при добавлении фото сразу открывалась камера, чтобы быстро сфотографировать банку без лишних действий.

#### Acceptance Criteria

1. WHEN THE User нажимает кнопку добавления фото в Post_Creation_System, THE Camera_Module SHALL открыть камеру устройства
2. THE Camera_Module SHALL настроить камеру на захват квадратного изображения с соотношением сторон 1:1
3. WHEN THE User делает снимок через Camera_Module, THE Post_Creation_System SHALL автоматически применить кроп 1:1 к захваченному изображению
4. THE Camera_Module SHALL отображать рамку предпросмотра с соотношением сторон 1:1 перед захватом
5. WHEN THE User отменяет захват фото, THE Post_Creation_System SHALL вернуться к экрану создания поста без добавления изображения
6. THE Post_Creation_System SHALL поддерживать добавление до 6 фотографий через последовательные вызовы Camera_Module

### Requirement 4: Group Selection Memory

**User Story:** Как пользователь, я хочу, чтобы при создании поста автоматически выбиралась группа, в которую я публиковал последний раз, чтобы не выбирать её вручную каждый раз.

#### Acceptance Criteria

1. WHEN THE User создаёт пост и выбирает группу, THE Group_Memory_Service SHALL сохранить идентификатор выбранной группы
2. WHEN THE User открывает экран создания поста, THE Post_Creation_System SHALL запросить у Group_Memory_Service последнюю выбранную группу
3. IF Group_Memory_Service возвращает сохранённую группу, THEN THE Post_Creation_System SHALL автоматически установить эту группу как выбранную
4. IF сохранённая группа больше не существует или User не является её членом, THEN THE Post_Creation_System SHALL очистить сохранённое значение и не устанавливать группу
5. THE Group_Memory_Service SHALL сохранять выбор группы локально на устройстве
6. WHEN THE User успешно публикует пост, THE Group_Memory_Service SHALL обновить сохранённую группу на группу текущего поста
7. THE Post_Creation_System SHALL позволять User изменить автоматически выбранную группу вручную

### Requirement 5: Multi-Group Feed Query Optimization

**User Story:** Как пользователь, я хочу, чтобы лента загружалась быстро даже при участии в большом количестве групп, чтобы не ждать долго при открытии приложения.

#### Acceptance Criteria

1. WHEN THE User состоит в более чем 10 группах, THE Feed_System SHALL ограничить параллельные запросы до 10 одновременных соединений
2. THE Feed_System SHALL использовать пул соединений для управления параллельными запросами к Firestore
3. WHEN THE Feed_System получает посты из группы, THE Feed_System SHALL объединять результаты в общий буфер
4. THE Feed_System SHALL применять сортировку по дате только после получения всех порций из всех групп
5. IF запрос к одной из групп завершается ошибкой, THEN THE Feed_System SHALL продолжить загрузку постов из остальных групп
6. THE Feed_System SHALL отображать частичные результаты User во время загрузки остальных данных

### Requirement 6: Camera Permission Handling

**User Story:** Как пользователь, я хочу получить понятное объяснение, если приложению нужен доступ к камере, чтобы понимать, зачем это необходимо и как предоставить разрешение.

#### Acceptance Criteria

1. WHEN THE Camera_Module требует доступ к камере, THE Post_Creation_System SHALL запросить разрешение у операционной системы
2. IF User отклоняет разрешение на доступ к камере, THEN THE Post_Creation_System SHALL отобразить сообщение с объяснением необходимости доступа
3. THE Post_Creation_System SHALL предоставить кнопку для открытия настроек приложения для предоставления разрешения
4. IF разрешение на камеру уже предоставлено, THEN THE Camera_Module SHALL открыть камеру без дополнительных запросов
5. WHEN THE User предоставляет разрешение на камеру, THE Camera_Module SHALL немедленно открыть камеру

### Requirement 7: Image Quality and Compression

**User Story:** Как пользователь, я хочу, чтобы фотографии загружались быстро и не занимали много места, чтобы экономить трафик и хранилище.

#### Acceptance Criteria

1. WHEN THE Camera_Module захватывает фото, THE Post_Creation_System SHALL применить сжатие JPEG с качеством 85
2. THE Post_Creation_System SHALL ограничить максимальный размер изображения до 1600 пикселей по большей стороне
3. THE Post_Creation_System SHALL сохранить соотношение сторон 1:1 после сжатия
4. WHEN THE Post_Creation_System сжимает изображение, THE Post_Creation_System SHALL отображать индикатор прогресса
5. THE Post_Creation_System SHALL проверять размер сжатого файла и отклонять изображения больше 5 МБ

### Requirement 8: Feed Real-Time Updates

**User Story:** Как пользователь, я хочу видеть новые посты в ленте в реальном времени, чтобы быть в курсе последних публикаций в моих группах.

#### Acceptance Criteria

1. THE Feed_System SHALL подписаться на real-time обновления постов из групп User
2. WHEN новый пост создаётся в одной из групп User, THE Feed_System SHALL автоматически добавить его в начало ленты
3. WHEN пост удаляется из группы, THE Feed_System SHALL автоматически удалить его из ленты
4. WHEN пост обновляется, THE Feed_System SHALL автоматически обновить его данные в ленте
5. THE Feed_System SHALL анимировать появление новых постов в ленте
6. THE Feed_System SHALL сохранять позицию прокрутки User при добавлении новых постов в начало ленты

### Requirement 9: Empty State Handling

**User Story:** Как пользователь, я хочу видеть понятное сообщение, когда в ленте нет постов, чтобы понимать, что делать дальше.

#### Acceptance Criteria

1. WHEN THE Feed_System загружает ленту и не находит постов, THE Feed_System SHALL отобразить сообщение о пустой ленте
2. THE Feed_System SHALL предложить User вступить в группы или создать первый пост
3. THE Feed_System SHALL отображать кнопку для перехода к списку групп
4. THE Feed_System SHALL отображать кнопку для создания нового поста
5. WHEN THE User не состоит ни в одной группе, THE Feed_System SHALL отобразить специальное сообщение с призывом вступить в группы

### Requirement 10: Offline Support

**User Story:** Как пользователь, я хочу видеть ранее загруженные посты даже без интернета, чтобы просматривать контент в любых условиях.

#### Acceptance Criteria

1. THE Feed_System SHALL кэшировать загруженные посты локально на устройстве
2. WHEN THE User открывает ленту без подключения к интернету, THE Feed_System SHALL отобразить кэшированные посты
3. THE Feed_System SHALL отображать индикатор offline-режима в верхней части ленты
4. WHEN подключение к интернету восстанавливается, THE Feed_System SHALL автоматически обновить ленту
5. THE Feed_System SHALL синхронизировать локальный кэш с сервером при восстановлении подключения
6. THE Post_Creation_System SHALL сохранять черновики постов локально при отсутствии подключения
