# Project Structure

## Repository Layout

```
banka/
├── lib/                      # Dart source code
│   ├── app/                  # Root app widget
│   ├── core/                 # Shared infrastructure
│   ├── features/             # Feature modules (Clean Architecture)
│   ├── firebase_options.dart # Generated Firebase config
│   └── main.dart             # Entry point
├── test/                     # Unit tests (mirrors lib/)
├── android/                  # Android native code
├── ios/                      # iOS native code
├── functions/                # Cloud Functions (Node 20)
├── firestore.rules           # Firestore Security Rules
├── firestore.indexes.json    # Composite indexes
├── firebase.json             # Firebase deploy config
└── [docs]                    # PROJECT_PLAN.md, CONTRIBUTING.md, etc.
```

```
│   └── failures.dart         # Failure sealed class

│   └── app_router.dart       # go_router configuration
├── theme/                    # UI theming
│   ├── app_colors.dart       # Color palette (#000000, #FFB300, etc.)
│   └── app_theme.dart        # Material 3 theme
├── usecases/                 # Base usecase contracts
│   └── usecase.dart          # UseCase<Type, Params>
└── utils/                    # Utilities
    └── typedefs.dart         # ResultFuture, ResultStream, etc.
```

## Feature Structure (Clean Architecture)

Each feature follows strict Clean Architecture layers:

```
features/<feature>/
├── data/                     # Data layer (external concerns)
│   ├── datasources/          # Remote/local data sources
│   │   ├── <feature>_remote_datasource.dart      # Abstract contract
│   │   └── firestore_<feature>_remote_datasource.dart  # Implementation
│   ├── models/               # DTOs (Data Transfer Objects)
│   │   └── <model>_dto.dart  # fromMap/toMap, fromSnapshot
│   └── repositories/         # Repository implementations
│       └── <feature>_repository_impl.dart  # Maps exceptions → Failures
├── domain/                   # Business logic (pure Dart)
│   ├── entities/             # Domain models (freezed)
│   │   └── <entity>.dart
│   ├── repositories/         # Repository contracts (abstract)
│   │   └── <feature>_repository.dart
│   └── usecases/             # Use cases (one per file)
│       ├── get_<entity>.dart
│       ├── watch_<entity>.dart
│       └── create_<entity>.dart
└── presentation/             # UI layer
    ├── bloc/                 # BLoC state management
    │   ├── <feature>_bloc.dart
    │   ├── <feature>_event.dart   # Sealed events
    │   └── <feature>_state.dart   # Equatable state
    ├── pages/                # Full-screen pages
    │   └── <page>_page.dart
    └── widgets/              # Reusable widgets
        └── <widget>.dart
```

## Key Conventions

### File Naming

- **Entities**: `<entity>.dart` (e.g., `post.dart`)
- **DTOs**: `<entity>_dto.dart` (e.g., `post_dto.dart`)
- **Repositories**: `<feature>_repository.dart` (contract), `<feature>_repository_impl.dart` (impl)
- **Usecases**: `<verb>_<entity>.dart` (e.g., `create_post.dart`, `watch_posts_feed.dart`)
- **BLoCs**: `<feature>_bloc.dart`, `<feature>_event.dart`, `<feature>_state.dart`
- **Pages**: `<page>_page.dart` (e.g., `profile_page.dart`)
- **Widgets**: `<widget>.dart` (e.g., `post_card.dart`)

### Import Organization

1. Dart SDK imports
2. Flutter imports
3. Package imports (alphabetical)
4. Relative imports (alphabetical)

### Generated Files (DO NOT EDIT)

- `*.freezed.dart` — Generated by freezed
- `*.g.dart` — Generated by json_serializable
- `lib/core/di/injector.config.dart` — Generated by injectable
- `lib/firebase_options.dart` — Generated by flutterfire_cli

These files are in `.gitignore` and regenerated via `dart run build_runner build`.

## Architecture Patterns

### Data Flow

```
UI (Widget)
  ↓ dispatch event
BLoC
  ↓ call
UseCase
  ↓ call
Repository (interface)
  ↓ implements
RepositoryImpl
  ↓ call
DataSource (interface)
  ↓ implements
FirestoreDataSource
  ↓ Firestore SDK
```

### Error Handling

```
Firestore Exception
  ↓ catch in DataSource
ServerException / NotFoundException
  ↓ catch in RepositoryImpl
Either<Failure, T>
  ↓ return to UseCase
Either<Failure, T>
  ↓ fold in BLoC
State (success / error)
  ↓ BlocBuilder
UI (show data / error message)
```

### Dependency Injection

- **Annotation**: `@injectable`, `@lazySingleton`, `@LazySingleton(as: Interface)`
- **Registration**: Automatic via `injectable_generator`
- **Access**: `getIt<Type>()` or constructor injection in other injectables
- **Setup**: `configureDependencies()` in `main.dart`
